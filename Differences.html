The DIFFERENCES between HTTP and HTTPS

DIFFERENCES HTTP HTTPS
URL Scheme "http://" "https://"
Data Encryption No encrytion provided SSL/TLS in data transmission
Port Port 80 Port 433
Authentication No build in mechanisms Digital certificates
Security Susceptible Secured/encrted

URL Scheme

To visit the website of HTTP and HTTPS, they have different URL formats.

For HTTP, the link of the http website must start from “ http://“. A client will request the http from the server, and the server responds with the corresponding HTTP to the client.

For the HTTPS, the link will start from “https://“. Similar to HTTP, a client sends an HTTPS request to the server,but before transferring the data, the client and the server establish a secured and encrypted connection.

Data Encrytion

HTTP does not provide encryption in data transfer. It transmits the data in plain text.

HTTPS provides SSL (Secure Sockets Layer)/TLS (Transport Layer Security) encryption which encrypts the data into encrypted formats instead of plain text to protect the confidentiality and integrity of data.

Port

HTTP uses port 80 as the default port for data communications

While HTTPS makes use of port 443 as the default port for communication.

Authentication

HTTP does not provide any built in authentication mechanisms.

HTTPS provides a mechanism for clients to verify the identity of the server. The mechanism is done through digital certificates issued by trusted Certificates Authorities (CAs).

The SIMILARITIES between HTTP and HTTPS

SIMILARITIES HTTP & HTTPS
Request Method GET/POST/PUT/DELETE
Client-server Model Request-response model
Resource Locating Uniform Resource Identifiers (URIs)
Communication Application Layer Protocol

Request Method

Both HTTP and HTTPS use the same request methods to interact with web resources.

Common methods include GET (retrieve a resource), POST (submit data to be processed), PUT (store or update a resource), and DELETE (remove a resource).

These method allows clients to interact with the server and perform actions such as retrieving, creating, updating resources.

Client-server Model

Both HTTP AND HTTPS follow a client-server model, where the client initiates a request, and the server responds with the requested content or performs the requested action

Resource Locating

Both HTTP and HTTPS use Uniform Resource Identifiers (URIs) to identify and locate web resources on the internet. URIs can be used to specify the location of web pages, imagesm documents, or any other resource that can be accessed via a URL.

Communication

Both HTTP and HTTPS are application layer protocols used for communication between a client (such as a web browser) and a server
